Significance Test — Complete Guide

Proportion Z-Test  ·  Mean Z-Test  ·  Z vs T-Table  ·  SmartSurvey CTable

What is a Significance Test?

A significance test (also called a hypothesis test) answers the question:

"Is the difference between two numbers large enough to be real — or could it be due to random chance?"

In survey research, we compare values between column groups (e.g. Product A vs Product B, Dhaka vs Rajshahi). Even when two percentages look different on paper, the difference might simply be due to sampling variation. The sig test quantifies how confident we are that the observed difference is genuine.

  • If the difference is statistically significant → the result is written as an UPPERCASE letter in the sig test row (e.g. B means this column is significantly higher than column B).
  • If significant at the lower confidence level only (when two levels are selected) → written as a lowercase letter (e.g. b).
  • If the difference is not significant → the cell is left blank.

Test 1 — Proportion Z-Test

Used for   Percentage rows: Yes/No, Agree/Disagree, TOP 2 BOX, BOTTOM 2 BOX, and all other % data rows.

The Formula

Proportion Z-Test:
z  =  |p₁ − p₂|  /  √( p₁(1−p₁)/n₁  +  p₂(1−p₂)/n₂ )

Variable Definitions

VariableMeaningWhere it comes from
p₁, p₂Proportions for column 1 and column 2 (percentage ÷ 100)Data row (e.g. "Yes" row), divided by 100
n₁, n₂Base sample sizes for each columnTotal row (the row labelled "Total" just below the letter row)
zComputed test statisticCompared against the critical value to decide significance

Step-by-Step Calculation

Example: Column A (55%, n=63) vs Column B (78%, n=64)

  1. Convert percentages to proportions:   p₁ = 55/100 = 0.55  ·  p₂ = 78/100 = 0.78
  2. Compute variance for each:   p₁(1−p₁)/n₁ = 0.55 × 0.45 / 63 = 0.003929  ·  p₂(1−p₂)/n₂ = 0.78 × 0.22 / 64 = 0.002681
  3. Sum the variances:   0.003929 + 0.002681 = 0.006610
  4. Take the square root:   √0.006610 = 0.08131
  5. Compute z:   |0.55 − 0.78| / 0.08131 = 0.23 / 0.08131 = 2.829
  6. Compare: 2.829 > 1.960 (95% critical value) → Significant at 95% → write uppercase A in Column B's S.TEST cell.

When the Formula Returns Zero

  • If either n = 0 → no comparison possible, result = 0.
  • If the denominator = 0 (both proportions are exactly 0% or 100%) → no variance to measure, result = 0.

Test 2 — Mean Z-Test

Used for   The MEAN row only. The mean z-test uses Standard Error (S.E.) values instead of sample proportions.

Why a Different Formula?

The proportion z-test only works for data that is either 0 or 1 (yes/no, agree/disagree). A mean score (e.g. 4.2 on a 5-point scale) is a continuous value — the proportion formula would produce nonsensical results. Instead we use the standard error of the mean to quantify uncertainty.

The Formula

Mean Z-Test using Standard Error:
z  =  |mean₁ − mean₂|  /  √( SE₁²  +  SE₂² )

Variable Definitions

VariableMeaningWhere it comes from
mean₁, mean₂Mean scores for column 1 and column 2MEAN row (directly above SIG. TEST)
SE₁, SE₂Standard Error of the mean for each columnS.E. row (2 rows below SIG. TEST)
zComputed test statisticCompared against critical value

Step-by-Step Calculation

Example: Column A (mean=4.38, SE=0.0829) vs Column B (mean=4.53, SE=0.0738)

  1. Difference of means:   |4.38 − 4.53| = 0.15
  2. Square each SE:   SE₁² = 0.0829² = 0.006872  ·  SE₂² = 0.0738² = 0.005446
  3. Sum of squares:   0.006872 + 0.005446 = 0.012318
  4. Square root:   √0.012318 = 0.11098
  5. Compute z:   0.15 / 0.11098 = 1.351
  6. Compare: 1.351 < 1.645 (90%) → Not significant at 90% → cell left blank.
Why use SE and not SD?
Standard Deviation (S.D.) measures spread within a single group. Standard Error (SE = SD / √n) measures the precision of the mean estimate — it accounts for sample size. Comparing means using SE directly is mathematically equivalent to using SD/√n, but simpler since SE values are already present in the table.

Z-Test vs T-Table

Both the proportion test and the mean test compute a z-score. The question is: what critical value do we compare it against?

Method Critical Value Source Accounts for Sample Size? Best For
Z-Test Standard Normal distribution (fixed values: 1.96, 1.645…) No — same threshold for n=30 or n=3000 Large samples (n > 30 per column)
T-Table t-distribution with df = n₁ + n₂ − 2 (adjusts per pair) Yes — threshold is higher when samples are small Small or moderate samples; more conservative

How the T Critical Value is Calculated

The t critical value is derived from the z critical value using the Cornish-Fisher series expansion:

Cornish-Fisher expansion (df = n₁ + n₂ − 2):
t  =  z  +  (z³ + z) / (4·df)  +  (5z⁵ + 16z³ + 3z) / (96·df²)  +  (3z⁷ + 19z⁵ + 17z³ − 15z) / (384·df³)

This converges to z as df → ∞ (large samples). Examples at 95% confidence (z = 1.960):

Important: With T-Table selected, the threshold is computed per column pair because each pair has its own n₁ and n₂. A pair of small-n columns gets a higher (stricter) threshold than a pair of large-n columns in the same table.

Group Definition File

The Group Definition file tells the sig test which columns should be compared against each other. Only columns within the same group line are compared — columns in different lines are never compared, avoiding meaningless cross-group comparisons.

File Format

3,4,5,6,7,8,9          ← Group 1 column numbers  (Product: C3–C9)
10,11                   ← Group 2 column numbers  (Panel 1: C10–C11)
12,13                   ← Group 3 column numbers  (Panel 2: C12–C13)
14,15                   ← Group 4 column numbers  (Panel 3: C14–C15)
...
26,27,28,29             ← Group N column numbers  (Centre – Dhaka)
30,31,32,33             ← Group N+1 column numbers (Centre – Rajshahi)
############################
ABCDEFG                 ← Maximum Set 1 letters used (A–O range)
PQRSTUV                 ← Maximum Set 2 letters used (P–Z range)
  • Above ####: Each line is one comparison group. Column numbers are absolute Excel column indices (C3 = 3, C10 = 10, etc.).
  • Below ####: Reference showing the maximum letters assigned from each letter set. Used for documentation only — the sig test uses the column numbers above.

Why Column Numbers, Not Letters?

Letters repeat across groups (e.g. both Column 3 and Column 12 may have letter A in the letter row — Product-A and Panel2-A respectively). If the sig test compared by letter, it would incorrectly cross-compare Product columns with Panel columns. Column numbers are always unique and unambiguous.

Calculations

Group 1: C3 (A=40%, n=50) vs C4 (B=70%, n=50)

p₁ = 0.40, p₂ = 0.70

z = |0.40 − 0.70| / √(0.40×0.60/50 + 0.70×0.30/50)

z = 0.30 / √(0.0048 + 0.0042) = 0.30 / √0.0090 = 0.30 / 0.0949 = 3.16

3.16 > 1.960 → Significant at 95% → write uppercase A in Column B's S.TEST cell.

Group 2: C5 (P=55%, n=50) vs C6 (Q=65%, n=50)

p₁ = 0.55, p₂ = 0.65

z = |0.55 − 0.65| / √(0.55×0.45/50 + 0.65×0.35/50)

z = 0.10 / √(0.00495 + 0.00455) = 0.10 / √0.0095 = 0.10 / 0.09747 = 1.026

1.026 < 1.645 → Not significant at 90% or above → cell left blank.

Cross-group comparison is blocked:
C3 (Group 1) and C5 (Group 2) are in different group definition lines. Even though both have value 40% and 55%, they are never compared against each other. This is correct — Product A and Panel 1 are completely different categories.
