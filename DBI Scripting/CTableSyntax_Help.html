Create CTable Syntax

Complete reference guide for generating SPSS recode and CTABLES syntax from the structure Excel file.

1 Overview

The Create CTable Syntax form generates ready-to-run SPSS .SPS files for data preparation (Recode) and custom table production (CTABLES). All analysis specifications are driven by a structured Excel file so you only need to fill the spreadsheet — the tool writes the syntax for you.

Four types of syntax can be generated:
  • Recode Syntax — Declares banner variables (NUMERIC / COMPUTE / RECODE / IF), adds mean/SD/SE variables for scaled questions, and saves the recoded dataset.
  • Table Cpt Syntax — CTABLES with column percentage (COLPCT) format.
  • Table Cpt (%) Syntax — CTABLES with column percentage using PCT8.0 display format.
  • Table Count Syntax — CTABLES with raw frequency counts.
💡 Always run Recode Syntax first in SPSS before running any table syntax, since the table syntax reads the recoded dataset.

2 Step-by-Step Workflow

1
Browse SPSS Data File

Select the cleaned SPSS .sav dataset. This is used to read variable value labels when generating banner syntax.

2
Create Structure File

Click Create Structure File. The tool reads the .sav and generates SPSS Analysis strcture.xlsx in the same folder, pre-populated with all variables from the dataset.

3
Complete the Structure Excel

Open the generated Excel file and fill in the Table Structure sheet (Analysis Type codes, MR groups, filters) and the Banner Variables sheet (define banner variables). See Sections 4 and 5 for full details.

4
Browse Structure Excel

Select the completed structure Excel file. The worksheet list on the left will populate automatically. Select the worksheet to process.

5
Set Up the Banner File

Click Get Banner File to open an example banner file for reference. Create your own banner .txt file following the format in Section 6, then browse and select it. The Output File Name is auto-filled from line 3 of the banner file.

6
Select Analysis Type & Syntax Type

Choose Unweighted or Weighted Analysis. Select a syntax type radio button. Adjust Decimal Place if needed (applies to Table Cpt only).

7
Execute

Click Execute to generate the .SPS file. It is saved in the same folder as the Structure Excel. Open it in SPSS Statistics and run.

4 Table Structure Sheet

Each row defines one SPSS variable and how it appears in the generated syntax. The sheet is auto-populated by Create Structure File — review and adjust each row before running Execute.

Col Column Name Description Fill
A Analysis Type Numeric code (0–12) that determines how the variable is tabulated. Auto-filled based on variable name patterns — always review. See Section 8 for the full code reference. Required
B Variable Name SPSS variable name. Auto-filled from the dataset. Do not change. Auto
C Variable Type SPSS variable type (e.g. Numeric, String). Auto-filled. String/Text variables are automatically excluded (set to code 0). Auto
D Variable Label The variable label from SPSS. Auto-filled. Used as the table title in CTABLES output. Auto
E MR Variable Name For Multiple Response (code 2): the MRSETS group name (without the $ prefix), e.g. Q3Brands. Fill on the last variable of the MR group only. Auto-filled by Create Structure File. MR only
F Break Point For Multiple Response: enter XXX on the last variable of each MR group. This triggers the MRSETS definition and the CTABLES table output for that group. MR only
G MR Label The label for the Multiple Response set, used as the CTABLES table title. Auto-filled from the Variable Label of the last MR variable. MR only
H Filter Condition An SPSS SELECT IF expression applied before this table only (e.g. DEM_GENDER = 1). Leave blank for no filter — the table runs on all respondents. Optional
I Filter Label Text description of the filter shown in the CTABLES corner cell (e.g. Males Only). Defaults to All Respondents if left blank. Optional

Setting Up a Multiple Response (MR) Group

For a multiple-select question (e.g. variables Q3_1, Q3_2, Q3_3):

  1. Set Analysis Type = 2 for every variable in the group.
  2. On the last variable only (e.g. Q3_3):
    • Column E → the group name, e.g. Q3Brands
    • Column F → XXX
    • Column G → the MR set label, e.g. Brand Awareness

Applying a Filter to a Specific Table

Enter a valid SPSS logical expression in Column H. The filter is applied with TEMPORARY. SELECT IF ... so it only affects that one table.

Column H example: DEM_GENDER = 1 Column I example: Males Only Generated syntax: TEMPORARY. SELECT IF DEM_GENDER = 1. CTABLES ... /TITLES CORNER='Base : Males Only'.
⚠ Create Structure File auto-fills Analysis Type codes based on variable name patterns. Always review them — especially Multiple Response groups and scaled questions which may need manual correction.

5 Banner Variables Sheet

This sheet defines the banner (column) variables used across all CTABLES output. It supports three modes depending on how each banner variable is derived.

Col Column Name Description
A SPSS Variables Source SPSS variable name(s). A single name for Mode 1 & 2 (e.g. DEM_CENTRE). Space-separated list for Mode 3 (e.g. DEM_GENDER DEM_AGE).
B Banner Variables The new derived banner variable name that will appear in CTABLES (e.g. b1Centre). Must be a valid SPSS variable name.
C Banner Labels The VARIABLE LABEL for the banner variable (e.g. Centre). If left blank, no VARIABLE LABELS statement is generated for this variable.
D New Code The new numeric value for this group (e.g. 1). Must be a whole number. Leave blank for Mode 1 (direct copy).
E New Label The VALUE LABEL for the new code (e.g. 18-34). Required when New Code is filled.
F Condition Mode 2: Source values as a space-separated list (1 2 3) or SPSS range notation (18 THRU 34).
Mode 3: Full SPSS IF expression (DEM_GENDER=1 AND DEM_AGE>=18 AND DEM_AGE<=34).
⚠ New Code, New Label and Condition must all be filled or all be blank in any given row. Partial fills will trigger a validation error and abort execution.

Mode 1 — Direct Copy  Columns A–C only, D–F empty

Copies the source SPSS variable directly into the new banner variable using COMPUTE. Value labels are pulled automatically from the SPSS dataset — no manual entry needed.

Validation Rules

  • New Code, New Label and Condition must all be filled or all be blank in each row.
  • Rows for the same Banner Variable cannot mix filled and blank New Code rows.
  • New Code values must be unique within each Banner Variable group.
  • For Mode 2 (RECODE), New Code must be a whole number.
  • If no custom grouping is used (Mode 1), the SPSS variable must exist in the selected dataset.
  • Multiple SPSS variables in Column A require at least one custom grouping row (Mode 3 only).

6 Banner File Format

The Banner File is a plain .txt file with exactly three data lines. Lines beginning with * are treated as comments and ignored.

Line Content Purpose in Generated Syntax Example
Line 1 Banner ColVar Used in CTABLES TABLE ... BY <ColVar> clause. Combines variables with + and supports nesting with >. ATotal+b1Centre+b1Age
Line 2 Banner ColVarPct Used in CTABLES CATEGORIES VARIABLES=<ColVarPct> clause. Space-separated list of all banner variable names. ATotal b1Centre b1Age
Line 3 Output File Name Project name used in the output .xlsx file path inside the syntax and in SPSS GET FILE / SAVE OUTFILE paths. ProjectName

Example Banner File

* Banner (ColVar) ATotal+b1Centre+b1Age+b1SEC * Banner (ColVarPct) ATotal b1Centre b1Age b1SEC * Output File Name MyProject_Wave1

Nested Banner Example

Use > to create nested (hierarchical) banners in CTABLES:

* Line 1 — nested banner: ATotal+b1Gender+(b1AgeM+b1AgeF)>b1Gender * Line 2 — all variables space-separated: ATotal b1Gender b1AgeM b1AgeF
💡 Click Get Banner File on the form to generate a ready-to-use example file that opens directly in Notepad.
⚠ The banner variable names in Lines 1 and 2 must exactly match the names defined in the Banner Variables sheet (Column B).

8 Analysis Type Code Reference

Set these codes in Column A of the Table Structure sheet. The Code Mapping sheet in the structure Excel is also a quick reference.
